# 1. Introduction

This roadmap outlines the development plan for Project Atlas Version 1.

The goal is to build the application in small, manageable stages while ensuring each feature is tested before moving to the next.

---

# 2. Development Strategy

Project Atlas will follow an incremental development approach.

Each sprint delivers a working part of the application.

No sprint begins until the previous sprint has been completed and verified.

---

# 4. Sprint 1 – Project Setup

Objectives

- Create repository
- Configure Electron
- Configure React
- Configure SQLite
- Create project structure
- Configure Git
- Verify development environment

Deliverable

A clean project ready for development.

---

# 5. Sprint 2 – Database & Backend

Objectives

- Create database
- Create database tables
- Build repositories
- Implement CRUD operations
- Verify data storage

Deliverable

A fully functional local database.

---

# 6. Sprint 3 – Browser Extension

Objectives

- Create Chrome extension
- Detect supported websites
- Collect business information
- Ask for user confirmation
- Send data to desktop application

Supported Sources

- Google Maps
- Google Search
- Company Websites

Deliverable

Working browser extension.

---

# 7. Sprint 4 – Desktop Application

Objectives

- Build Dashboard
- Build Leads page
- Build Projects page
- Build Categories page
- Build Settings page
- Build Search

Deliverable

Functional desktop interface.

---

# 8. Sprint 5 – Lead Processing

Objectives

- Validation
- Duplicate detection
- Lead scoring
- Category assignment
- Project conversion
- Export (CSV & JSON)

Deliverable

Complete lead management system.

---

# 9. Sprint 6 – Testing & Release

Objectives

- Fix bugs
- Improve performance
- Test all features
- Prepare installer
- Final documentation review

Deliverable

Project Atlas Version 1.0 Release Candidate.

---

# 10. Release Checklist

Before releasing Version 1, verify:

- Browser Extension works correctly
- Desktop Application starts successfully
- Database stores data correctly
- Lead scoring is accurate
- Duplicate detection works
- Search functions correctly
- Export works
- No critical bugs remain
- Documentation is complete

---

# Roadmap Summary

The Project Atlas roadmap breaks development into six focused sprints.

Each sprint builds on the previous one, resulting in a stable, maintainable, and fully functional Version 1 release.
